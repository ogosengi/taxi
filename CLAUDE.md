## 핵심 아키텍처

### 모델 구조
```csharp
TaxiWorkShift    // 근무시간 정보 (유연한 시간 설정)
DailySettlement  // 일별 마감 정보 (매출 및 통계)
TaxiOperationStats // 운행 현황 통계
```

### 데이터베이스 스키마
```sql
-- 근무시간 테이블 (매출 컬럼 제거됨)
CREATE TABLE WorkShifts (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    Date TEXT NOT NULL,
    StartTime TEXT NOT NULL,
    EndTime TEXT NOT NULL,
    IsNightShift INTEGER NOT NULL,
    Notes TEXT
);

-- 일별마감 테이블 (최적화됨)
CREATE TABLE DailySettlements (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    Date TEXT NOT NULL UNIQUE,
    TotalRevenue REAL NOT NULL,
    TotalWorkingHours REAL NOT NULL,
    Notes TEXT
);
```

## 최신 업데이트 (2025-09-17)
### 🎯 주요 개선사항
- **근무시간 입력 최적화**:

  - 탭 순서: 월 → 일 → 시작시간 → 종료시간 → 야간근무 → 메모 → 추가
  - 매출조회 탭순서: 마감일 → 매출 → 일별마감

  - 팝업 제거: 근무시간 추가, 일별마감, 마감자료 조회 시 알림 팝업 제거
  - 레이아웃 개선: 메모 입력란 옆으로 추가 버튼 이동
  - 그리드 스크롤: 추가한 데이터가 정확히 보이도록 자동 스크롤
- **데이터 관리 개선**:
  - 그리드 정렬: 날짜 오름차순 정렬
  - 시간당 평균: 소수점 1자리 표시
  - 마감처리 일시 필드 제거 (DB/UI 모두)
  - 근무시간 입력 시 마감일 자동 대입
- **통계 정보 개선**:

  - 완료된 근무 관련 항목 제거
  - 더 직관적이고 간결한 통계 정보
- **데이터베이스 마이그레이션**:
  - 기존 데이터 보존하면서 스키마 변경

### 🔬 정밀한 효율성 분석 계산식
시간대별 효율성은 실제 근무시각을 기준으로 다음과 같이 계산됩니다:

1. **근무시간 분할**:
   - 각 근무를 시간대별로 정확히 분할
   - 예: 10:30-14:30 근무 → 10시(0.5h), 11시(1h), 12시(1h), 13시(1h), 14시(0.5h)
   - 야간근무도 다음날로 넘어가는 시간을 정확히 계산

2. **매출 비례 배분**:
   - 일별 총 매출을 해당 날짜의 모든 근무시간에 비례하여 배분
   - 각 근무의 시간대별 세그먼트에 근무시간 비율로 매출 재배분

3. **시간대별 집계**:
   - 모든 근무의 시간대별 세그먼트를 합산
   - 실제 그 시간대에 근무한 총 시간과 배분된 총 매출 계산

4. **효율성 계산**:
   - 시간당 평균 매출 = 시간대별 총 매출 / 시간대별 총 근무시간
   - 24시간 전체 패턴으로 가장 효율적인 시간대 식별
